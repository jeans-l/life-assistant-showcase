## 1. 核心判断

每一次生产力革命，都会同时改变两件事：一是供给如何变得更充裕，二是用户需求与供给之间的匹配方式如何被重构。工业革命让商品大规模生产，信息革命让信息大规模数字化，而 AI 革命正在把“获取信息之后如何做出决策”这条链路重新改写。

生活助理的愿景，就建立在这个判断上：AI 的价值不应只停留在搜索、整理和推荐，而应该进一步进入 `decision-making / execution`，替普通人承担一部分原本只能靠外部专业判断才能完成的复杂生活决策。

## 2. 富人的生活方式为什么重要

主项目愿景里有一个很重要的观察：富人的效率，不只是拥有更多资源，更重要的是他们可以持续购买专业判断。

厨师、教练、税务顾问、律师、职业经理人，这些角色背后的共同机制不是“替你定义人生目标”，而是：

1. 目标仍由用户自己定义。
2. 复杂情境中的判断可以委托给更专业的系统。
3. 在高风险、高个性化节点，用户仍保留最终授权权。

生活助理要做的，不是复制某一个专业角色，而是把“持续获得专业判断”这件事，从少数人的高成本配置变成大众可获得的基础设施。

## 3. AI 时代的普惠机会

如果说搜索和推荐解决的是“信息太多，我找不过来”，那么生活助理要解决的是“约束太多、节奏太碎、判断太频繁，我不想每次都从零开始决定”。

因此，AI 助理的价值不应只停留在：

1. 帮我搜索
2. 帮我整理
3. 帮我推荐

而应升级到：

1. 理解我的长期偏好与稳定约束
2. 在具体场景中替我做默认决策
3. 在低风险任务上继续推进执行
4. 在结果反馈中持续学习，让后续判断越来越稳定

## 4. 产品定位

生活助理的定位，不是单一场景工具，也不是一个更会说话的 AI。它是一个以个人生活为服务对象的助理系统。

这意味着产品核心不是“对话自然不自然”，而是：

1. 能不能了解我是谁
2. 能不能记得我怎么生活
3. 能不能在合适的时间，帮我做合适的安排

从这个角度看，聊天只可能是入口，不是产品本体。生活助理真正想重做的，是“基于 memory 的主动决策与执行”。

## 5. 第一性原理

主项目愿景把底层链路收得很清楚：

`目标 -> 约束 -> 决策 -> 执行 -> 反馈 -> 学习`

其中：

1. 目标由用户定义
2. 约束来自偏好、预算、时间、健康、环境与历史行为
3. 决策由 AI 助理完成
4. 执行由系统或外部服务承接
5. 反馈进入记忆与后续判断

这也是生活助理和普通聊天机器人最根本的分界线。后者可以给出回答，但前者必须形成一条可持续学习、可持续承接的服务链。

## 6. 为什么不是聊天

从愿景层看，生活助理不想做“一个更会聊天的 AI”，而想做“一个全心全意为个人服务的助手”。

用户保留目标定义权，系统承担默认判断，关键节点保留最终授权，这三者必须同时成立。只要系统把所有复杂度都重新扔回给用户，它就仍然只是一个会回答问题的界面，而不是助理。

## 7. 最终愿景

让每个人都拥有一个可被信任的个人生活助理：

1. 节约精力：替用户承担越来越多高频、低风险、重复性的生活决策
2. 尊重用户：在关键节点保留用户的目标定义权和最终授权权
3. 持续学习：在执行和反馈中逐步摸清边界，让默认判断越来越可靠

主项目最后给出的愿景很直接：让判断力与精力的延展，从富人的专属配置，变成大众基础设施。
